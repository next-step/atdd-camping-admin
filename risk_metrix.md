# 리스크 매트릭스 (Risk Matrix)

## 평가 기준 (1-5 척도)

| 평가 항목 | 가중치 | 설명 |
|----------|--------|------|
| 비즈니스 임팩트 | 40% | 장애 발생 시 매출/고객 신뢰 영향 |
| 버그 발생 가능성 | 30% | 현재 코드의 취약점 수준 |
| 변경 빈도 | 20% | 향후 수정될 가능성 |
| 기술적 복잡도 | 10% | 구현 복잡성, 의존성 |

## 기능별 상세 평가

### 1. PATCH /admin/reservations/{id}/status (예약 상태 변경)

| 항목 | 점수 | 근거 |
|------|------|------|
| 비즈니스 임팩트 | **5** | 예약 취소/확정은 매출과 직결, 고객 불만 유발 가능 |
| 버그 발생 가능성 | **4** | 상태값 검증 없음, 상태 전이 규칙 부재 |
| 변경 빈도 | **4** | 환불/알림 등 후속 로직 추가 예상 |
| 기술적 복잡도 | **3** | Map 파싱, 상태 전이 로직 필요 |

**종합 점수**: 5×0.4 + 4×0.3 + 4×0.2 + 3×0.1 = **4.3** (High)

---

### 2. POST /admin/rentals (렌탈 등록)

| 항목 | 점수 | 근거 |
|------|------|------|

| 버그 발생 가능성 | **3** | DTO 사용으로 양호하나 재고 차감 동시성 이슈 가능 |
| 변경 빈도 | **3** | 반납, 연장 기능 추가 예상 |
| 기술적 복잡도 | **4** | 재고 차감, 상품 타입 검증, 트랜잭션 관리 |

**종합 점수**: 4×0.4 + 3×0.3 + 3×0.2 + 4×0.1 = **3.5** (Medium-High)

## 테스트 우선순위 추천

### P0 (즉시) - 예약 상태 변경

```gherkin
# 반드시 먼저 작성
1. 정상 취소 플로우 (CONFIRMED → CANCELLED)
2. 존재하지 않는 예약 ID 처리
3. 잘못된 상태값 요청 시 검증 (현재 버그 존재)
```

### P1 (높음) - 렌탈 등록

```gherkin
1. 정상 렌탈 등록 (예약 연결/미연결)
2. RENTAL 타입 아닌 상품으로 시도 시 거부 확인
3. 재고 부족 상황 처리
4. 재고 차감 정확성 검증
```

### P2 (중간) - 상품 등록

```gherkin
1. 정상 상품 등록
2. 필수 필드(name) 누락 시 에러 응답
3. 음수 가격/재고 거부 확인 (현재 버그 존재)
```

### P3 (낮음) - 캠핑장 사이트 / 예약 조회

```gherkin
# 캠핑장 사이트
1. 정상 등록
2. siteNumber 중복 시 에러

# 예약 조회
1. 예약 있을 때 목록 반환
2. 예약 없을 때 빈 배열 반환
```
