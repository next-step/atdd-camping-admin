## 경험 공유하기

1. 인수 테스트 기반으로 리팩터링을 하면서 어떤 점을 체감했는지 작성하기
    - 좀 더 마음 편하게 리팩토링을 진행할 수 있었습니다.
      - 이후 postman이나 curl을 호출하지 않고도 검증할 수 있었습니다.
      - 단위 테스트만 작성하는 경우에도 외부 호출이 좀 달라질 수 있어 postman으로 검증을 다시 진행하는 경우들이 많았는데 그럴 필요 x
    - 단위 테스트 내용과 인수 테스트 내용에 중복들이 존재해 다시 검증해야 하나? 라는 생각도 들었습니다.
      - 작성해야 할 테스트가 2배로 늘어난 느낌?
2. 리팩터링 과정에서 테스트를 수정했는지, 했다면 왜 했는지를 작성하기 
   - enum 같은 것으로 리팩토링 시 인수 테스트가 수정되는 경우가 많아짐
     - Map을 사용해서 검증하지 말고 타입을 명확하게 기존 DTO를 사용하는 것이 이런 면에서는 더 좋을 수도 있겠다는 생각이 들었습니다.
     - value object로 리팩톨이 시에서 인수 테스트가 함께 수정되어야 하는 문제 발생
3. AI 도구를 활용하는 경우 방법과 느낀점을 작성하기
   - md 파일과 /docs로 프로젝트 문서 관리
   - ATDD 방식과 기존 api 에 대해 인수 테스트하는 방식을 나눠서 md 파일로 관리
   - 아직 잘 사용하지는 못해서 좀 더 고민이 필요해보입니다
     - 더 빠르고 정확하게 할 수 있게 문서 작성 공부 필요
