# 인수 조건 (Acceptance Criteria)

## 1. 대여 생성 (POST /admin/rentals) - 리스크 1위

### Happy Path
**인수 조건**: "관리자가 RENTAL 타입 상품을 예약 고객에게 대여하면 재고가 차감되고 대여 기록이 생성된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 201 Created이다
- [ ] 응답에 대여 기록 ID가 포함된다
- [ ] 상품 재고가 요청 수량만큼 감소한다
- [ ] 대여 기록의 isReturned가 false이다
- [ ] 대여 기록에 예약 ID가 연결된다

---

### 예외 케이스 1: 재고 부족
**인수 조건**: "재고보다 많은 수량을 대여 요청하면 실패한다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 에러 메시지에 재고 부족 안내가 포함된다
- [ ] 상품 재고가 변경되지 않는다
- [ ] 대여 기록이 생성되지 않는다

---

### 예외 케이스 2: SALE 타입 상품 대여 시도
**인수 조건**: "SALE 타입 상품을 대여 요청하면 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 에러 메시지에 'RENTAL 타입만 대여 가능' 안내가 포함된다
- [ ] 대여 기록이 생성되지 않는다

---

### 예외 케이스 3: 존재하지 않는 상품
**인수 조건**: "존재하지 않는 상품 ID로 대여 요청하면 실패한다"

**검증 포인트**:
- [ ] 응답 상태 코드가 404 Not Found이다
- [ ] 에러 메시지에 상품을 찾을 수 없음이 포함된다

---

### 엣지 케이스 1: 수량이 0 또는 음수
**인수 조건**: "대여 수량이 0 이하이면 요청이 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 상품 재고가 변경되지 않는다
- [ ] 대여 기록이 생성되지 않는다

---

### 엣지 케이스 2: 동시 대여 요청
**인수 조건**: "동시에 같은 상품을 대여 요청해도 재고 정합성이 유지된다"

**검증 포인트**:
- [ ] 재고가 음수가 되지 않는다
- [ ] 성공한 요청 수 × 수량 = 감소한 재고량이다
- [ ] 재고 부족 시 일부 요청은 실패한다

---

### 엣지 케이스 3: 예약 없이 대여 (워크인)
**인수 조건**: "예약 ID 없이 대여 요청하면 워크인 대여로 기록된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 201 Created이다
- [ ] 대여 기록의 reservationId가 null이다
- [ ] 상품 재고가 정상 차감된다

---

## 2. 예약 상태 변경 (PATCH /admin/reservations/{id}/status) - 리스크 2위

### Happy Path
**인수 조건**: "관리자가 예약 상태를 CANCELLED로 변경하면 해당 사이트는 다시 예약 가능하다"

**검증 포인트**:
- [ ] 응답 상태 코드가 200 OK이다
- [ ] 응답의 status가 CANCELLED이다
- [ ] 해당 날짜/사이트에 새 예약이 가능하다

---

### 예외 케이스 1: 유효하지 않은 상태 값
**인수 조건**: "정의되지 않은 상태 값으로 변경 요청하면 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 에러 메시지에 유효한 상태 목록이 포함된다
- [ ] 예약 상태가 변경되지 않는다

---

### 예외 케이스 2: 존재하지 않는 예약
**인수 조건**: "존재하지 않는 예약 ID로 상태 변경 요청하면 실패한다"

**검증 포인트**:
- [ ] 응답 상태 코드가 404 Not Found이다
- [ ] 에러 메시지에 예약을 찾을 수 없음이 포함된다

---

### 예외 케이스 3: 빈 요청 본문
**인수 조건**: "빈 요청 본문으로 상태 변경 요청하면 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 예약 상태가 변경되지 않는다

---

### 엣지 케이스 1: 허용되지 않는 상태 전이
**인수 조건**: "CANCELLED 상태의 예약을 CONFIRMED로 변경하면 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 에러 메시지에 상태 전이 불가 안내가 포함된다
- [ ] 예약 상태가 CANCELLED로 유지된다

---

### 엣지 케이스 2: null 또는 빈 문자열 상태
**인수 조건**: "status가 null이거나 빈 문자열이면 요청이 거부된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 400 Bad Request이다
- [ ] 예약 상태가 변경되지 않는다

## 5. 예약 목록 조회 (GET /admin/reservations) - 리스크 5위

### Happy Path
**인수 조건**: "관리자가 예약 목록을 조회하면 모든 예약이 반환된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 200 OK이다
- [ ] 응답이 배열 형태이다
- [ ] 각 예약에 id, customerName, startDate, endDate, status가 포함된다

---

### 엣지 케이스 1: 예약이 없는 경우
**인수 조건**: "예약이 없으면 빈 배열이 반환된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 200 OK이다
- [ ] 응답이 빈 배열 `[]`이다

---

### 엣지 케이스 2: 대량 데이터 조회
**인수 조건**: "예약이 많아도 정상적으로 조회된다"

**검증 포인트**:
- [ ] 응답 상태 코드가 200 OK이다
- [ ] 응답 시간이 허용 범위 내이다
- [ ] 모든 예약이 반환된다
