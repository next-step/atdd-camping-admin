# 1. 앱 기동 방식 차이

## 임베디드 - @SpringBootTest

테스트 실행 → Spring이 앱을 자동으로 띄움 (랜덤 포트) → 테스트 → 앱 종료
- 테스트가 알아서 앱을 띄우고 끔
- 포트가 매번 바뀜 (@LocalServerPort로 받아옴)

## 별도 프로세스

1. 터미널에서 ./gradlew bootRun (앱 수동 실행, 8080 고정)
2. 다른 터미널에서 ./gradlew test (테스트만 실행)
- 앱이 이미 떠있다고 가정하고 테스트
- 포트 8080 고정
