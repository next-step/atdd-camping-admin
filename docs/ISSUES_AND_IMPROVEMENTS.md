## 리스크 매트릭스

### 평가 기준

| 항목 | 1점 | 2점 | 3점 |
|------|-----|-----|-----|
| **영향도 (Impact)** | 낮음: 불편함 수준 | 중간: 기능 오류 | 높음: 데이터 손실/보안 위협 |
| **발생 가능성 (Likelihood)** | 낮음: 특수 상황 | 중간: 간헐적 발생 | 높음: 일상적 발생 가능 |

**리스크 점수** = 영향도 x 발생 가능성 (1~9점)

## 우선순위별 상세

### P0 - 즉시 수정 (점수 6~9)

#### #1. 재고 감소 Race Condition (9점)
- **문제**: Read-Check-Write 패턴으로 동시 요청 시 재고 정합성 깨짐
- **해결**: 비관적 락 또는 원자적 UPDATE 쿼리 적용

#### #2. 평문 비밀번호 비교 (6점)
- **문제**: 설정 파일에 평문 저장, 메모리 노출 위험
- **해결**: PasswordEncoder + BCrypt 해시 적용

#### #3. N+1 쿼리 + OOM 위험 (6점)
- **문제**: 전체 데이터 메모리 로드 + Lazy Loading으로 쿼리 폭발
- **해결**: JPQL 집계 쿼리로 DB 레벨 계산

---

### P1 - 단기 개선 (점수 4~5)

#### #4. 반납 처리 Race Condition (4점)
- **문제**: 중복 반납 가능성
- **해결**: 낙관적 락 또는 상태 검증 쿼리

#### #5. JSON Injection (4점)
- **문제**: 에러 메시지에 특수문자 포함 시 JSON 깨짐
- **해결**: ObjectMapper 사용

#### #6. 상태값 검증 없음 (4점)
- **문제**: 임의의 문자열이 상태값으로 저장 가능
- **해결**: enum 검증 로직 추가
