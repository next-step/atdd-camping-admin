# Admin API 리스크 매트릭스

## 평가 기준
- **비즈니스 임팩트**: 1(낮음) ~ 5(치명적)
- **변경 빈도**: 1(거의 없음) ~ 5(매우 빈번)
- **버그 가능성**: 1(낮음) ~ 5(매우 높음)
- **총점**: 3개 항목 합계 (최대 15점)

## 평가 결과

## Top 3 선정 근거
1. **예약 상태 변경 (14점)**: 상태 검증 로직 부재로 임의 문자열 허용 시 잘못된 예약 취소/확정으로 직접적인 매출 손실과 고객 신뢰도 하락 위험
2. **대여 생성 (11점)**: 재고 확인과 차감 간 원자성 보장 로직의 부재로 동시 요청 시 재고 오버부킹과 운영 혼란 발생 위험  
3. **상품 생성 (11점)**: 수동 파싱에서 예외 발생 시 0원으로 설정된 상품이 등록되어 매출 손실 위험
