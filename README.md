## 테스트 구조
### 테스트 러너 클래스
테스트 러너 클래스는 테스트를 실행하는 진입점 역할을 하는 클래스입니다.  
Cucumber에서 테스트를 실행할 때 어떤 feature 파일들을 실행하고, 어떤 step definitions를 사용할지, 어떤 설정을 적용할지를 정의하는 클래스입니다.
#### 왜 필요한가?
* 테스트 실행 진입점: 테스트를 시작하는 시작점 역할  
* 설정 관리: Cucumber의 다양한 설정을 한 곳에서 관리  
* 테스트 그룹화: 관련된 테스트들을 그룹화하여 실행  
* CI/CD 통합: 자동화된 테스트 실행을 위한 표준화된 진입점  

## IntelliJ에서 인수테스트 실행시 breakpoint 안 걸리는 이유 및 해결법

### 💡 IDE에서 디버깅이 안 걸리는 이유와 해결책

핵심 문제:
현재 Cucumber 테스트는 RestAssured를 사용해서 HTTP 요청을 보내는 구조입니다.  
이는 외부 프로세스로 실행되는 Spring Boot 애플리케이션과 통신하는 방식이므로,  
IDE에서 TestRunner만 디버그 실행해서는 브레이크포인트가 걸리지 않습니다.

### 🎯 해결 방법 : Spring Boot 애플리케이션을 별도로 디버그 모드로 실행

- 1단계: 애플리케이션을 디버그 모드로 실행
    - > ./gradlew bootRun --debug-jvm

- 2단계: IDE에서 Remote Debug 연결
    - > IntelliJ: Run → Attach to Process → localhost:5005  
      또는 Run → Edit Configurations → Add → Remote JVM Debug

- 3단계: 브레이크포인트 설정 후 Cucumber 테스트 실행
    - `RentalService.createRental()`에 브레이크포인트 설정
    - IDE에서 TestRunner 실행

## 🌼 과제 진행하면서 느낀 경험 🌼

### 1. 인수 테스트 기반으로 리팩터링을 하면서 체감한 내용

- **세분화된 커밋단위**
    - 하나의 인수 테스트 내에서도`given, when, then` 별로, commit 단위를 작게 가져갈 수 있었습니다.
- **리팩토링 과정의 안정감**
    - 도메인 단위의 단위 테스트와 다르게 하나의 기능에 대해 완벽히 동작함이 시나리오별로 검증되어있어 리팩토링에 안정감이 더 크게 있었습니다.
        - 예를 들어, `대여시 재고가 줄어든다` 정책은 `대여(Rental)` 도메인과 `상품(Product)` 도메인 둘다 검증이 필요한데, 두 도메인을 인수테스트에서 한번에 다루고 있어 시나리오를 완벽히
          커버했다고 확신할 수 있었습니다.

### 2. 리팩터링 과정에서 테스트를 수정했는지, 했다면 왜 했는지를 작성하기

- 수정사항
    - 입력값의 유효성에 대해 검증하는 부분이 있었는데, 인수 테스트보다는 Controller 테스트로 분리하는 것이 맞겠다는 판단이 들었습니다.
- 이유
    - 특정 포멧(null)의 입력값이 유효하다는것은 정책이기보다는 기술적 제약조건이라는 생각이 들었습니다.
- 기타
    - 마침 비슷한 [고민 내용](https://nextstep-members.slack.com/archives/C09BY9CCTPB/p1758452469208229)이 슬랙에 올라와서 참고했습니다 ㅎㅎ

### 3. AI 도구를 활용하는 경우 방법과 느낀점을 작성하기

- 명확한 프롬프트가 필요하다
    - `예약 업데이트` api 에 대해 테스트 커버리지 100에 달하도록 시나리오를 만들어달라고 요청하니, 조건만 약간 다른 중복 시나리오가 나왔었습니다. (유효하지 않은 문자를 입력한 경우에 대해 5개 가량
      중복 시나리오 도출) 처음부터 중복을 막아야한다고 프롬프트를 입력해야겠다고 느꼈습니다.
- 예시코드를 잘 만들어야한다.
    - 제가 작성한 RentalSteps.java 를 보고 ai가 시나리오 기반으로 UpdateReservationSteps.java 를 작성했는데 구현 방식이 비슷했습니다. 그런데 코드가 깔끔하지 않은 부분도
      그대로 따라하는 경향이 있어, 초기단계 코드 구조를 잘 잡아야겠다는 생각이 들었습니다.
