## 인수 테스트 기반 리팩터링을 하면서 체감한 점
- 인수 테스트가 코드의 안정성을 크게 높여준다는 것. 리팩터링 과정에서 기존 기능이 제대로 작동하는지 확인할 수 있어, 안심하고 코드를 개선할 수 있었다.
- 테스트 작성이 처음에는 시간이 걸리지만, 장기적으로 보면 디버깅 시간과 유지보수 비용을 절감할 수 있다는 것.
- 테스트 코드가 잘 작성되어 있으면, 새로운 기능 추가나 변경이 필요할 때도 기존 기능에 영향을 미치지 않는지 쉽게 확인할 수 있다는 것.
- 리팩터링 과정에서 코드의 가독성과 유지보수성을 높이는 데 집중하게 되어, 코드 품질이 전반적으로 향상된다는 것.

## 리팩터링 과정에서 테스트를 수정했는지, 했다면 그 이유를 작성한다.
- 리팩터링 과정에서 일부 테스트를 수정했다. 
  - 코드 구조가 변경되면서 일부 테스트 케이스가 더 이상 유효하지 않거나, 새로운 구조에 맞게 조정이 필요했기 때문.
  - 리팩터링으로 인해 기존 테스트가 더 이상 충분하지 않다고 판단되어 추가적인 테스트 케이스를 작성했다.
  - 테스트 코드 자체의 가독성과 유지보수성을 높이기 위해, 중복된 코드를 제거하고, 테스트의 목적을 명확히 하기 위해 리팩터링했다.

## AI 도구를 활용했다면 방법과 느낀 점을 작성한다.
- AI 도구를 활용하여 코드 리팩터링과 테스트 작성에 도움을 받았다. 예를 들어, 코드 스니펫 생성, 코드 리뷰, 테스트 케이스 제안 등의 기능을 사용했다.
- AI 도구가 반복적이고 시간이 많이 소요되는 작업을 자동화해주어, 개발자가 더 중요한 문제에 집중할 수 있게 해준다는 것.
- AI 도구를 활용하면서도 최종 결정은 개발자가 내려야 한다는 점을 항상 염두에 두어야 한다는 것을 깨달았다. 
- 디버깅의 경험이 별로였다. AI가 제안한 코드가 항상 최적의 솔루션이 아니기 때문에, 개발자가 직접 코드를 검토하고 테스트하는 과정이 여전히 중요하다는 것을 느꼈다.
