# 리스크 매트릭스

## 평가 기준
- **척도**: 높음(3점), 보통(2점), 낮음(1점)
- **계산식**: 리스크 점수 = 비즈니스 임팩트 + 변경 빈도 + 버그 가능성

## Top 3 선정 근거
### 1위: 예약 상태 변경 (총점 9점)
- 비즈니스 임팩트 (3점): 고객 예약의 직접적인 상태 변경으로 매출과 고객 만족도에 즉각적 영향
- 변경 빈도 (3점): 일상적인 예약 관리 업무로 가장 빈번하게 사용되는 기능
- 버그 가능성 (3점): 상태 검증 로직의 단순함으로 인한 잘못된 상태값 입력 위험

### 2위: 대여 생성 (총점 8점)
- 비즈니스 임팩트 (3점): 재고 관리와 직결되어 재고 부족 시 매출 손실 발생 가능
- 변경 빈도 (2점): 캠핑 시즌과 날씨에 따라 변동적이지만 정기적으로 발생
- 버그 가능성 (3점): 복잡한 재고 감소 로직과 비즈니스 규칙으로 인한 높은 오류 가능성

### 3위: 예약 목록 조회 (총점 6점)
- 비즈니스 임팩트 (2점): 조회 기능이므로 직접적인 매출 영향은 제한적이나 운영 효율성에 중요
- 변경 빈도 (2점): 대시보드와 관리 화면에서 지속적으로 사용되는 핵심 기능
- 버그 가능성 (2점): 필터링 로직의 복잡성과 null 처리로 인한 중간 수준의 위험
