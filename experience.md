# 경험 공유하기

## 인수 테스트 기반으로 리팩터링을 하면서 어떤 점을 체감했는지 작성하기

인수 테스트 기반으로 리팩터링을 하며, 테스트와 로직의 동작보다도  
현재 작성된 "인수조건" 과 "요구사항"이 무엇인지에 대해서  
계속 생각을 할 수 있어서 실제 제품을 만드는데 있어서 무척 유용하다고 체감했습니다.  
클린코드, 객체지향, TDD 에 집중하다보면 그래서 결국 고객이 무엇을 원하고  
요구사항이 정확히 무엇이었는지에 대해서 지속적으로 생각하지 못하는데,  
성능 개선이 아니라 고객 만족을 위한 고민을 개발을 하며 계속 할 수 있었다는 점에서  
매우 유용한 개발 방법론이라 생각합니다.

## 리팩터링 과정에서 테스트를 수정했는지, 했다면 왜 했는지를 작성하기

리팩터링 과정에서 예외처리 관련된 테스트를 무척 많이 수정했습니다.  
성공 시나리오의 경우에는 명확한 요구사항이므로 오히려 수정할 부분이 적었습니다.  
반면 사용자가 잘못된 값을 입력했을 경우에 대해서는,  
사용자가 편하려면 시스템이 자동적으로 값을 교정해주는 게 좋을지,  
아니면 동작을 멈추도록 해서 사용자에게 본인이 입력한 값을 직접 확인하도록 유도할지,  
무엇이 사용자에게 더 유익할지 고민하고 그에 따른 테스트를 수정하게 되었습니다.

## AI 도구를 활용하는 경우 방법과 느낀점을 작성하기

AI 도구는 Claude Code 를 사용했습니다.  
느낀점은 결국 AI 는 본인이 할 수 있는 일을 빠르게 더 많이 해줄 뿐이라는 점이었습니다.  
AI 도구로 작성된 코드는 더 빠르게 작성되었을 뿐, 제가 직접 손으로 작성하더라도 작성이 가능할 것 같습니다.  
다르게 생각하면 만약 제가 적절치 못한 설계와 개발 습관을 가지고 있다면, 저의 명령에 따르는 AI 또한 적절치 못한 결과물들을 잔뜩 만들어낼 것 같습니다.  
개발팀 관점에서 본다면 개발 부채가 더 빠르게 쌓일 수 있는 위험도 있겠다는 생각이 들었습니다.  
AI 도구 때문에 개발팀의 개발 부채가 더 빠르게 쌓이기 때문에 테스트 코드가 더더욱 중요해 지는 시대가 올 것 같습니다.
