# 경험 공유하기

## 인수 테스트 기반으로 리팩터링하며 느낀점
리팩터링 하는 과정에서 내가 수정한 코드가 원래 구현된 코드와 결과가 달라지지 않았을까 걱정되는 부분이 있었습니다. \
인수 테스트를 작성하며 리팩터링 하면 결과가 바뀌게 될 경우 테스트를 통해 미리 확인하고 대응할 수 있어서 안정적인 리팩터링이 가능했습니다. \
또한 문제가 생겼을때 인수 테스트로 검증된 기능은 문제가 없다 생각하고 다른 부분에서 문제를 빠르게 찾아낼 수 있었습니다.

## AI 도구를 활용하는 경우 방법과 느낀점
### 사용 방법
1. jdbcTemplate 같이 사용법을 잘 모를 경우
2. 단순 반복 작업이 필요한 경우
3. 예상치 못한 에러가 발생한 경우

### 느낀점
1. 사용법을 익히고 감을 잡을 필요 없이 AI 에게 요구사항을 얘기하면 알아서 만들어줘서 새로운 언어나 환경이 두렵지 않아짐
2. 어떻게 구현해야될지 감이 잡히지만 하드 코딩된 부분이 있거나 단순 반복 작업이 필요해 귀찮은 작업을 대신 해줘서 편함
3. 대신 에러 코드를 분석해 문제 원인, 해결방법 까지 알려줘서 구글링에 사용할 시간을 많이 절약할 수 있음
