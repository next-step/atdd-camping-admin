# 인수 조건 정의서

> 리스크 평가 기준 고위험 API 순서대로 5W1H 분석 및 인수 조건 정의

---

## 1. 예약 상태 변경 (리스크 점수: 14)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 관리자 |
| **What** (무엇을) | 고객의 예약 상태를 변경한다 |
| **When** (언제) | 고객 요청 또는 운영 상황에 따라 |
| **Where** (어디서) | 관리자 페이지에서 |
| **Why** (왜) | 예약 확정, 취소, 체크인/아웃 처리를 위해 |
| **How** (어떻게) | 예약 목록에서 해당 예약을 선택하고 상태를 변경한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-1.1 | 관리자가 확정된 예약을 취소하면, 예약 상태가 "취소"로 변경되어야 한다. |
| AC-1.2 | 관리자가 대기 중인 예약을 확정하면, 예약 상태가 "확정"으로 변경되어야 한다. |
| AC-1.3 | 관리자가 확정된 예약을 체크인 처리하면, 예약 상태가 "체크인"으로 변경되어야 한다. |
| AC-1.4 | 존재하지 않는 예약을 변경하려고 하면, 오류 메시지가 표시되어야 한다. |
| AC-1.5 | 유효하지 않은 상태값으로 변경하려고 하면, 오류 메시지가 표시되어야 한다. |

## 2. 대여 생성 (리스크 점수: 12)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 관리자 |
| **What** (무엇을) | 고객에게 캠핑 장비를 대여해준다 |
| **When** (언제) | 고객이 체크인할 때 또는 현장 방문 시 |
| **Where** (어디서) | 캠핑장 관리 데스크에서 |
| **Why** (왜) | 고객이 필요한 장비를 제공하고 수익을 창출하기 위해 |
| **How** (어떻게) | 예약 정보와 상품을 선택하고, 대여 수량을 입력하여 등록한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-2.1 | 관리자가 예약 고객에게 장비를 대여하면, 해당 상품의 재고가 대여 수량만큼 감소해야 한다. |
| AC-2.2 | 예약 없이 현장 방문 고객에게도 장비를 대여할 수 있어야 한다. |
| AC-2.3 | 재고보다 많은 수량을 대여하려고 하면, "재고 부족" 오류가 표시되어야 한다. |
| AC-2.4 | 판매 전용 상품은 대여할 수 없고, 오류 메시지가 표시되어야 한다. |
| AC-2.5 | 존재하지 않는 상품을 대여하려고 하면, 오류 메시지가 표시되어야 한다. |

## 3. 판매 처리 (리스크 점수: 12)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 직원 또는 시스템 |
| **What** (무엇을) | 캠핑 용품을 고객에게 판매한다 |
| **When** (언제) | 고객이 물품 구매를 원할 때 |
| **Where** (어디서) | 캠핑장 매점 또는 관리 데스크에서 |
| **Why** (왜) | 고객에게 필요한 물품을 제공하고 매출을 발생시키기 위해 |
| **How** (어떻게) | 판매할 상품과 수량을 선택하여 결제를 완료한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-3.1 | 상품을 판매하면, 해당 상품의 재고가 판매 수량만큼 감소해야 한다. |
| AC-3.2 | 여러 상품을 한 번에 판매할 수 있어야 한다. |
| AC-3.3 | 재고보다 많은 수량을 판매하려고 하면, "재고 부족" 오류가 표시되어야 한다. |
| AC-3.4 | 존재하지 않는 상품을 판매하려고 하면, 오류 메시지가 표시되어야 한다. |
| AC-3.5 | 판매가 완료되면, 판매 기록이 저장되어야 한다. |

## 4. 대여 반납 (리스크 점수: 11)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 관리자 |
| **What** (무엇을) | 고객이 반납한 대여 장비를 처리한다 |
| **When** (언제) | 고객이 체크아웃할 때 또는 대여 기간 종료 시 |
| **Where** (어디서) | 캠핑장 관리 데스크에서 |
| **Why** (왜) | 반납된 장비를 재고에 복구하여 다른 고객에게 대여하기 위해 |
| **How** (어떻게) | 대여 목록에서 해당 대여 건을 선택하고 반납 처리한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-4.1 | 대여 장비를 반납 처리하면, 해당 상품의 재고가 대여 수량만큼 복구되어야 한다. |
| AC-4.2 | 반납 처리된 대여 건은 "반납 완료" 상태로 변경되어야 한다. |
| AC-4.3 | 이미 반납 처리된 대여 건을 다시 반납하려고 하면, 오류 메시지가 표시되어야 한다. |
| AC-4.4 | 존재하지 않는 대여 건을 반납하려고 하면, 오류 메시지가 표시되어야 한다. |

## 5. 수익 보고서 조회 (리스크 점수: 11)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 관리자 또는 경영진 |
| **What** (무엇을) | 캠핑장의 수익 현황을 확인한다 |
| **When** (언제) | 일일 마감 시 또는 경영 분석이 필요할 때 |
| **Where** (어디서) | 관리자 페이지의 보고서 메뉴에서 |
| **Why** (왜) | 매출 현황을 파악하고 경영 의사결정에 활용하기 위해 |
| **How** (어떻게) | 조회할 날짜 또는 기간을 선택하여 보고서를 생성한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-5.1 | 특정 날짜의 수익 보고서를 조회하면, 예약/판매/대여 수익이 각각 표시되어야 한다. |
| AC-5.2 | 특정 날짜의 총 수익은 예약+판매+대여 수익의 합계와 일치해야 한다. |
| AC-5.3 | 기간별 수익 보고서를 조회하면, 해당 기간의 누적 수익이 표시되어야 한다. |
| AC-5.4 | 데이터가 없는 날짜를 조회하면, 모든 수익이 0으로 표시되어야 한다. |
| AC-5.5 | 수익 상세 내역을 조회하면, 각 수익의 발생 일시와 항목이 표시되어야 한다. |

## 6. 로그인 (리스크 점수: 10)

### 5W1H 분석

| 항목 | 내용 |
|------|------|
| **Who** (누가) | 캠핑장 관리자 |
| **What** (무엇을) | 관리자 시스템에 로그인한다 |
| **When** (언제) | 관리 업무를 시작할 때 |
| **Where** (어디서) | 관리자 로그인 페이지에서 |
| **Why** (왜) | 관리자 전용 기능에 접근하기 위해 |
| **How** (어떻게) | 아이디와 비밀번호를 입력하여 인증한다 |

### 인수 조건

| ID | 인수 조건 |
|----|----------|
| AC-6.1 | 올바른 아이디와 비밀번호를 입력하면, 로그인에 성공하고 인증 토큰이 발급되어야 한다. |
| AC-6.2 | 잘못된 비밀번호를 입력하면, "인증 실패" 메시지가 표시되어야 한다. |
| AC-6.3 | 존재하지 않는 아이디를 입력하면, "인증 실패" 메시지가 표시되어야 한다. |
| AC-6.4 | 로그인하지 않고 관리자 페이지에 접근하면, 로그인 페이지로 이동해야 한다. |
